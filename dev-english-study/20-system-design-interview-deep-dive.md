# 后端深入：系统设计面试专项英语

## 1. write path
- 发音：短语表达
- 含义：写路径
- 例句：The write path should remain simple and reliable.
- 翻译：写路径应该保持简单且可靠。
- 讲解：系统设计高频表达。

## 2. read path
- 发音：短语表达
- 含义：读路径
- 例句：The read path can be optimized separately from the write path.
- 翻译：读路径可以与写路径分开优化。
- 讲解：读写分离常见表达。

## 3. bottleneck analysis
- 发音：短语表达
- 含义：瓶颈分析
- 例句：Bottleneck analysis should come before premature optimization.
- 翻译：在过早优化之前，应该先做瓶颈分析。
- 讲解：很适合面试回答。

## 4. capacity planning
- 发音：短语表达
- 含义：容量规划
- 例句：Capacity planning depends on traffic patterns and growth assumptions.
- 翻译：容量规划取决于流量模式和增长假设。
- 讲解：系统设计必备词汇。

## 5. workload pattern
- 发音：短语表达
- 含义：负载模式
- 例句：We need to understand the workload pattern before choosing a database.
- 翻译：在选择数据库前，我们需要先理解负载模式。
- 讲解：面试回答很加分。

## 6. hot key
- 发音：短语表达
- 含义：热点 key
- 例句：A hot key can overload a single partition.
- 翻译：热点 key 可能会压垮单个分区。
- 讲解：缓存和分布式存储常见。

## 7. request amplification
- 发音：短语表达
- 含义：请求放大
- 例句：A naive design may cause request amplification.
- 翻译：一个粗糙的设计可能造成请求放大。
- 讲解：系统复杂度讨论常见。

## 8. fault domain
- 发音：短语表达
- 含义：故障域
- 例句：Replicas should be distributed across different fault domains.
- 翻译：副本应该分布在不同的故障域中。
- 讲解：高可用设计高频概念。

## 9. blast radius
- 发音：短语表达
- 含义：故障影响范围
- 例句：We should reduce the blast radius of deployment failures.
- 翻译：我们应该降低部署失败带来的影响范围。
- 讲解：运维和架构复盘常见。

## 10. graceful degradation
- 发音：短语表达
- 含义：优雅降级
- 例句：Graceful degradation is better than complete failure.
- 翻译：优雅降级比完全失败更好。
- 讲解：真实系统设计思维。

## 11. read-after-write consistency
- 发音：短语表达
- 含义：写后读一致性
- 例句：User profiles usually require read-after-write consistency.
- 翻译：用户资料通常需要写后读一致性。
- 讲解：业务需求分析常用。

## 12. eventual convergence
- 发音：短语表达
- 含义：最终收敛
- 例句：Replicas should reach eventual convergence after temporary failures.
- 翻译：副本在临时故障后应该最终收敛一致。
- 讲解：分布式一致性相关。

## 13. write-heavy workload
- 发音：短语表达
- 含义：写密集型负载
- 例句：This storage engine is optimized for write-heavy workloads.
- 翻译：这个存储引擎针对写密集型负载做了优化。
- 讲解：数据库选择高频表达。

## 14. read-heavy workload
- 发音：短语表达
- 含义：读密集型负载
- 例句：A cache layer is especially useful for read-heavy workloads.
- 翻译：缓存层对读密集型负载尤其有帮助。
- 讲解：架构取舍常见。

## 15. cost efficiency
- 发音：短语表达
- 含义：成本效率
- 例句：The final design should balance reliability, performance, and cost efficiency.
- 翻译：最终设计应该平衡可靠性、性能和成本效率。
- 讲解：面试回答更完整、更像真实架构师。
