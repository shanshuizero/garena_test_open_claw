# 前端开发英语词汇

> 聚焦 HTML、CSS、JavaScript、框架、浏览器渲染与前端工程化。

---

## 1. layout
- 发音：/ˈleɪaʊt/
- 含义：布局
- 开发语境：页面结构、元素排布。
- 例句：The mobile layout breaks on small screens.
- 翻译：这个移动端布局在小屏设备上会错乱。
- 讲解：`breaks on small screens` 表示在小屏设备上出现布局问题。

## 2. responsive
- 发音：/rɪˈspɑːnsɪv/
- 含义：响应式的
- 开发语境：页面适配不同屏幕尺寸。
- 例句：We need a responsive design for tablets and desktops.
- 翻译：我们需要一个适配平板和桌面的响应式设计。
- 讲解：`responsive design` 是前端基础高频词组。

## 3. breakpoint
- 发音：/ˈbreɪkpɔɪnt/
- 含义：断点
- 开发语境：CSS 媒体查询里的尺寸阈值。
- 例句：Add another breakpoint for ultra-wide screens.
- 翻译：为超宽屏再增加一个断点。
- 讲解：`add another breakpoint` 是实际前端工作中的典型表达。

## 4. viewport
- 发音：/ˈvjuːpɔːrt/
- 含义：视口
- 开发语境：浏览器可视区域。
- 例句：The modal should stay centered in the viewport.
- 翻译：这个弹窗应该始终保持在视口中央。
- 讲解：`stay centered` 表示“保持居中”。

## 5. render
- 发音：/ˈrendər/
- 含义：渲染
- 开发语境：浏览器把数据和结构显示为界面。
- 例句：The page takes too long to render.
- 翻译：这个页面渲染耗时太长。
- 讲解：`takes too long to ...` 是非常实用的抱怨式表达。

## 6. hydration
- 发音：/haɪˈdreɪʃən/
- 含义：水合
- 开发语境：SSR 页面在客户端接管交互的过程。
- 例句：We are seeing hydration errors in the production build.
- 翻译：我们在生产构建中看到了水合错误。
- 讲解：这是 React / Next.js 生态高频术语。

## 7. bundle
- 发音：/ˈbʌndl/
- 含义：打包产物
- 开发语境：Webpack、Vite 等生成的 JS/CSS 文件。
- 例句：The main bundle is too large for first-time visitors.
- 翻译：主包对于首次访问用户来说太大了。
- 讲解：`first-time visitors` 指首次访问的用户。

## 8. asset
- 发音：/ˈæset/
- 含义：静态资源
- 开发语境：图片、字体、图标、脚本等。
- 例句：Static assets should be cached aggressively.
- 翻译：静态资源应该采用更激进的缓存策略。
- 讲解：`cached aggressively` 常见于性能优化场景。

## 9. selector
- 发音：/sɪˈlektər/
- 含义：选择器
- 开发语境：CSS 选择器、状态选择器。
- 例句：This selector is too specific and hard to override.
- 翻译：这个选择器太具体了，很难覆盖。
- 讲解：`override` 在 CSS 中表示“覆盖样式”。

## 10. specificity
- 发音：/ˌspesɪˈfɪsəti/
- 含义：选择器优先级
- 开发语境：CSS 规则冲突时谁生效。
- 例句：The styling issue is caused by selector specificity.
- 翻译：这个样式问题是由选择器优先级导致的。
- 讲解：`is caused by` 是定位原因的标准说法。

## 11. animation
- 发音：/ˌænɪˈmeɪʃən/
- 含义：动画
- 开发语境：CSS 或 JS 动画效果。
- 例句：The animation feels smooth on desktop but laggy on mobile.
- 翻译：这个动画在桌面端看起来很流畅，但在移动端会卡顿。
- 讲解：`smooth` 和 `laggy` 是很实用的体验描述词。

## 12. transition
- 发音：/trænˈzɪʃən/
- 含义：过渡
- 开发语境：状态变化时的样式过渡。
- 例句：Add a subtle transition to the hover state.
- 翻译：给悬停状态加一个轻微的过渡效果。
- 讲解：`subtle` 表示“轻微、不夸张”。

## 13. accessibility
- 发音：/əkˌsesəˈbɪləti/
- 含义：可访问性
- 开发语境：让残障用户也能访问和使用产品。
- 例句：We need to improve accessibility for keyboard users.
- 翻译：我们需要提升键盘用户的可访问性体验。
- 讲解：`keyboard users` 是无障碍讨论中的高频对象。

## 14. semantic
- 发音：/sɪˈmæntɪk/
- 含义：语义化的
- 开发语境：使用正确的 HTML 标签表达结构。
- 例句：Please use semantic HTML whenever possible.
- 翻译：请尽量使用语义化的 HTML。
- 讲解：`whenever possible` 表示“只要可能就”。

## 15. state
- 发音：/steɪt/
- 含义：状态
- 开发语境：组件状态、页面状态、全局状态。
- 例句：The loading state is not handled properly.
- 翻译：加载状态没有被正确处理。
- 讲解：`handled properly` 是评审代码时常见说法。

## 16. props
- 发音：/prɑːps/
- 含义：属性
- 开发语境：React 组件入参。
- 例句：Do not mutate props inside the child component.
- 翻译：不要在子组件内部修改 props。
- 讲解：`mutate` 是框架开发中的关键动词。

## 17. hook
- 发音：/hʊk/
- 含义：钩子
- 开发语境：React Hooks 或框架扩展点。
- 例句：This custom hook simplifies the fetch logic.
- 翻译：这个自定义 Hook 简化了获取数据的逻辑。
- 讲解：`custom hook` 是 React 生态高频词组。

## 18. event
- 发音：/ɪˈvent/
- 含义：事件
- 开发语境：点击、输入、滚动等交互事件。
- 例句：The click event is triggered twice on Safari.
- 翻译：这个点击事件在 Safari 上会触发两次。
- 讲解：`is triggered twice` 表示“被触发两次”。

## 19. routing
- 发音：/ˈruːtɪŋ/ 或 /ˈraʊtɪŋ/
- 含义：路由
- 开发语境：前端页面切换、URL 映射。
- 例句：Client-side routing improves the navigation experience.
- 翻译：客户端路由提升了导航体验。
- 讲解：`client-side routing` 是 SPA 常见表达。

## 20. debounce
- 发音：/diːˈbaʊns/
- 含义：防抖
- 开发语境：控制高频事件触发次数。
- 例句：We should debounce the search input.
- 翻译：我们应该对搜索输入做防抖处理。
- 讲解：`debounce the search input` 是面试与实战都很高频的表达。

## 21. throttle
- 发音：/ˈθrɑːtl/
- 含义：节流
- 开发语境：限制函数执行频率。
- 例句：Scroll events should be throttled for better performance.
- 翻译：为了更好的性能，滚动事件应该做节流处理。
- 讲解：`for better performance` 表示目的。

## 22. lazy loading
- 发音：/ˈleɪzi ˈloʊdɪŋ/
- 含义：懒加载
- 开发语境：按需加载组件、图片或模块。
- 例句：Lazy loading reduces the initial page load time.
- 翻译：懒加载可以减少页面初始加载时间。
- 讲解：`initial page load time` 是性能优化高频词组。

## 23. tree shaking
- 发音：/triː ˈʃeɪkɪŋ/
- 含义：摇树优化
- 开发语境：打包时移除未使用代码。
- 例句：Tree shaking does not work well with this library.
- 翻译：摇树优化在这个库上效果不好。
- 讲解：`does not work well with` 是兼容性讨论常见句式。

## 24. middleware
- 发音：/ˈmɪdlwer/
- 含义：中间件
- 开发语境：前端框架或状态管理中的中间处理层。
- 例句：The middleware logs every navigation event.
- 翻译：这个中间件会记录每一次导航事件。
- 讲解：虽然 middleware 常见于后端，但前端框架中也很常见。

## 25. re-render
- 发音：/ˌriːˈrendər/
- 含义：重新渲染
- 开发语境：状态更新后组件重复渲染。
- 例句：This state change causes unnecessary re-renders.
- 翻译：这个状态变化导致了不必要的重复渲染。
- 讲解：`unnecessary re-renders` 是 React 性能分析中的高频搭配。
