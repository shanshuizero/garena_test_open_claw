# Speaking Day 28：面试：系统设计 / 排障 / 好代码

## 晨读短句
1. I usually start by understanding the workload pattern.
   - 中文：我通常先理解负载模式。
   - 提示：系统设计回答

2. I usually start with metrics, logs, and recent changes.
   - 中文：我通常先看指标、日志和最近改动。
   - 提示：排障回答

3. Good code is not just code that works.
   - 中文：好代码不只是能运行的代码。
   - 提示：价值观题

4. Clarity is often more valuable than unnecessary cleverness.
   - 中文：清晰通常比炫技更有价值。
   - 提示：价值观题

5. I prioritize tasks based on business impact and technical risk.
   - 中文：我按影响和风险排优先级。
   - 提示：优先级题
